---

Key routing decisions to note:

- **Router** runs a cheap OCR-lite pass first and routes to one of 3 lanes вҖ” `form`, `card`, or `uncertain`. This avoids running the heavy YOLO orientation sweep on obvious application forms.

- **Form lane** skips YOLO entirely вҖ” just doc orientation correction + OCR pattern masking.

- **Card/uncertain lane** runs the full 3-model gate inside an orientation sweep loop (up to 8 angles), picks the best angle by score, then does OCR on Aadhaar crop regions only (not the full image).

- **Both services** (masking-engine and batch-processor) call the exact same `core.pipeline.process_image()` вҖ” zero code duplication.

- **DynamoDB** is only used by the batch-processor (status tracking: PENDING вҶ’ PROCESSING вҶ’ COMPLETED/ERROR). The API gateway path has no DB writes.
